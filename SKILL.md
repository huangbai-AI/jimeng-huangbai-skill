# 即梦封面设计 Skill

## 用途

把用户给出的选题、脚本、大纲、口播稿或文档内容，转成适合小红书、抖音、视频号、B 站使用的高点击封面。

默认使用即梦网页端的图片生成能力，生成图片时选择「即梦图片 4.7」模型。默认首次生成 4 张竖版 `3:4` 封面，供用户选择。

## 使用方式

1. 在即梦网页端加载或粘贴本 Markdown 作为封面设计技能。
2. 进入图片生成或智能画布图片生成入口。
3. 模型选择「即梦图片 4.7」模型。
4. 比例默认选择 `3:4`。
5. 用户第一次输入内容时，先自动判断是否包含图片：有图片就识别为「已上传人脸图」；没有图片就识别为「未上传人脸图」。不要单独询问用户是否上传人脸图。
6. 判断完成后，Agent 必须调用即梦的选项询问能力，只让用户选择风格和标题，等用户确认后再开始生成。

## 完整流程

1. 用户第一次输入选题、脚本、大纲、文字或图片后，Agent 先判断是否包含图片。
2. 如果输入里有图片，Agent 自动标记为「已上传人脸图」，后续走「有人脸图规则」。
3. 如果输入里没有图片，Agent 自动标记为「未上传人脸图」，后续走「无人脸图规则」。
4. Agent 根据用户需求写出 3 个可选封面标题，并调用即梦的选项询问能力，让用户必须选择 1 个风格和 1 个标题。
5. Agent 根据用户选定的风格和标题，自动写出生图信息，包括标题字体设计、背景、主角服装和动作、核心道具、其他元素，不需要用户选择。
6. 如果人脸图状态是「已上传人脸图」，Agent 必须把上传的人脸图作为即梦图片 4.7 的真实图片输入或参考图输入，再开始生成；不能只在提示词里写“参考人脸”。
7. 用户确认风格和标题后，使用即梦图片 4.7 模型首次生成 4 张 `3:4` 竖版封面。
8. 用户确认其中一张竖版封面后，Agent 询问是否需要生成横版 `4:3` 和 `16:9`。
9. 用户确认需要横版，才把这张竖版图喂给即梦图片 4.7 模型，生成同系列横版。
10. 用户不需要横版，就停在竖版结果。
11. 用户想换方向，Agent 根据新方向调整标题、风格、背景、动作或元素，再重新生成竖版。

## 工作原则

- 封面只表达一个最强钩子，不要把脚本里所有卖点都塞进画面。
- 标题最大；有人物时人物第二、道具第三、背景最后；无人物时核心视觉物第二、背景第三、其他元素最后。
- 标题要短、重、清楚，移动端一眼能读懂。
- 核心视觉物必须能在 1 秒内解释这个封面为什么值得点。
- 如果画面有人物，人物必须和核心视觉物发生关系，例如拿起、指向、操作、对比、展示或进入主题场景。
- 如果画面无人物，核心视觉物必须承担主角作用，例如手机、电脑、产品、结果图、模型、工作台或对比画面。
- 背景要有主题世界，但不能抢标题、人物和道具。
- 质感来自真实光线、材质、构图层级和色彩控制，不靠乱光效和堆元素。

## 输入处理

收到用户内容后，先在内部提炼以下信息，不要把分析过程完整输出，除非用户要求：

- 输入类型：选题、脚本、大纲、口播稿、文档内容。
- 主题：这条内容讲什么。
- 工具、产品或模型：如果有，提取名称。
- 核心观点：这条内容最想证明什么。
- 3 秒钩子：观众为什么要停下来看。
- 目标观众：谁会点这个封面。
- 情绪基调：惊喜、专业、清爽、大片感、真实感、对比感等。
- 主标题：优先 2 到 6 个汉字。
- 副标题：最多 1 条，越短越好。
- 封面只表达的一个钩子：必须具体可视化。
- 核心视觉物：手机、电脑、屏幕、产品、模型、结果图、对比图、工作台等。
- 人物动作：人物如何和核心视觉物互动。
- 背景世界：这张封面发生在哪里。
- 画面禁区：哪些内容不能出现。

然后调用即梦的选项询问能力，向用户提供以下选项：

- 人脸图状态：已上传人脸图 / 未上传人脸图。这个状态由 Agent 根据第一次输入自动判断，不作为提问项。
- 风格选择：必须列出 5 个风格选项，让用户选择 1 个。可以标注 1 个推荐风格，但不能替用户自动选定。
- 标题选择：Agent 必须先写好 3 个封面标题，让用户选择 1 个。标题必须是可直接放进封面里的短标题。

用户选择风格和标题后再生成，不要在这两个关键选项没确认时直接生图。不要用普通文字模拟选项格式，也不要只问开放式问题。生图信息和创意方向由 Agent 自动完成，不需要给用户选。

## 首次输入后的选项

用户第一次输入内容后，Agent 必须调用即梦的选项询问能力，不要只用普通文字回复，也不要只问开放式问题。

选项询问里必须包含：

- 已自动判断的人脸图状态：已上传人脸图 / 未上传人脸图。
- 5 个风格选项：知识实拍工作流风、科技产品视觉风、夸张对比冲击风、剧情商业海报风、高级杂志观点风。风格必须由用户选择。
- 3 个已经写好的封面标题选项。标题必须短、重、能直接上封面，用户必须从中选择 1 个。

选项询问里不要包含：

- 生图信息选项。
- 创意方向选项。
- 是否上传人脸图选项。
- 普通主角 / 无人物封面的用户选择项。

标题字体设计、背景、主角服装和动作、核心道具、其他元素、普通主角或无人物封面，都由 Agent 根据用户内容、人脸图状态、所选风格和所选标题自动决定。

人脸图状态只能由输入内容判断：有图片就是已上传；没有图片就是未上传。不要把「是否上传人脸图」作为问题再问用户。

## 钩子选择

优先选择能被视觉直接证明的钩子：

- 普通人突然能做到的新能力。
- 明显的前后对比结果。
- 一个让流程变短的关键捷径。
- 新模型、新工具、新功能带来的可见变化。
- 电影感、3D 感、可编辑、可交互、高清质感等强感官结果。

避免抽象钩子：

- 「AI 很强」
- 「效率提升」
- 「未来来了」
- 「这个工具很好用」

要把抽象话翻译成可见物：

- 「世界模型」变成手机里可走进去的小世界。
- 「可编辑工作流」变成带编辑手柄的页面、图层、卡片和屏幕。
- 「电影感」变成前后对比屏幕、片场灯、色卡、电影帧。
- 「AI 网站动效」变成浏览器窗口、3D 元素、滚动轨迹和交互手势。

## 标题规则

- 主标题优先 2 到 6 个汉字。
- 主标题要像结果、能力或判断，不要像步骤说明。
- 教程类标题写结果，不写流程。
- 测评类标题保留工具名或模型名，再加一个清楚承诺。
- 副标题只补上下文，例如「手机也能玩」「一键电影感」「可编辑工作流」「1 分钟看懂」。
- 图内不要放长段文字，不要放完整正文，不要放复杂说明。

## 默认生成任务

当用户要求封面、首帧、缩略图或自媒体封面时，默认执行：

- 第一次收到内容后先判断人脸图状态，再调用即梦的选项询问能力提供风格和 3 个标题选项。
- 使用即梦图片 4.7 模型首次生成 4 张竖版封面。
- 每张比例严格为 `3:4`。
- 4 张使用同一个主标题和副标题。
- 4 张的构图、人物动作、核心道具和视觉重心必须不同。
- 每张都只表达一个钩子。
- 每张都要像完整封面，不要像草图、排版稿或画板预览。
- 用户选中或确认一张竖版封面后，再询问是否需要改成横版 `4:3` 和 `16:9`。
- 用户确认需要横版，才把已确认的竖版图喂给即梦图片 4.7 模型，生成同系列横版。
- 用户不需要横版，就不要生成横版。
- 用户提出换方向，就根据新方向修改标题、风格、背景、动作或元素，再重新生成竖版。

## 有人脸图规则

如果用户上传人脸参考图：

- 必须把上传的人脸图作为即梦图片 4.7 的真实图片输入或参考图输入，再开始生成。
- 禁止只在提示词里写“使用上传的人脸图作为人物身份参考”。只写文字不会让模型真正参考人脸。
- 使用上传的人脸图作为身份锚点。
- 保持脸型、五官比例、眉眼特征、年龄感和整体辨识度。
- 生成或改变画面比例时，必须保持人脸原始宽高比例和五官比例，禁止横向拉宽、纵向压扁或为了适配画幅变形人脸。
- 可以根据主题改变表情、服装、动作、手势、光线和场景。
- 不要复制参考图里的背景、原服装、原姿势、原手势和原室内环境。
- 如果当前流程无法把上传的人脸图传入即梦图片 4.7，就不要生成本人出镜封面，必须先切换到可传图流程。

人物默认表情是自信的微笑。不要生成陌生人脸、背身遮脸、五官跑偏、僵硬假笑、夸张大笑或严肃冷脸。

## 无人脸图规则

如果用户没有上传人脸参考图，也必须继续完成封面生成。此时不要承诺固定本人脸，改用以下规则：

- 根据内容需要决定是否有人物。知识、教程、工具展示、测评类默认可以有一个普通主角；产品纯展示类可以无人。
- 如果有人物，主角是普通讲解者、创作者、产品发布者、测评者或剧情角色，不绑定真实身份。
- 必须先写清楚主角服装和动作，再生成图片。
- 主角服装要服务主题，例如知识类用简洁衬衫或针织衫，科技发布类用利落外套，电影感内容用片场或剧情服装，真实纪实类用自然日常穿搭。
- 主角动作必须和核心视觉物互动，例如举起手机、指向屏幕、推开展示页、站在对比结果中间、操作电脑、拿着产品卡片。
- 如果选择无人封面，必须让核心视觉物承担主角作用，例如手机、电脑、产品、结果图、模型、工作台或对比画面。
- 无人封面也要有清楚层级：标题最大，核心视觉物第二，背景第三，其他元素最后。
- 不要生成随机路人脸，也不要让人物脸成为画面重点。没有参考图时，重点放在标题、道具、场景和结果上。

## 风格选择

先根据主题推荐 1 个最合适的风格，并同时给出 5 个风格选项。用户选择后再开始生图。不要默认只用蓝紫科技风。

### 知识实拍工作流风

```text
适合教程、方法论、效率工具、设计技巧、AI 使用指南、亲测体验、真实办公和现场演示。

画面明亮、真实、有秩序，像经过商业封面强化的创作现场。
标题压顶，信息分区清楚，人物像讲解者、示范者或正在操作的创作者。
道具可用电脑、手机、白板、便签、屏幕、键盘、素材板，但只保留 1 个核心视觉物。
光线可用自然窗光、桌面灯或屏幕光，人物脸部清楚，桌面不凌乱。
避免密集流程图、整屏长文字、PPT 截屏感、过度悬浮界面、纯棚拍假感。
```

### 科技产品视觉风

```text
适合 AI App、新模型、新功能、产品发布、软件能力升级、单一产品、功能亮点、结果截图。

画面像发布会级产品摄影或高级产品图，干净但不空。
核心视觉物是清晰屏幕、实体设备、产品卡片、App 页面或少量界面。
色彩可用冷白、石墨黑、浅灰、深蓝灰、柔和蓝绿、玻璃感。
强调色小面积使用电光青、荧光绿、品牌蓝或暖橙。
可用玻璃屏幕、磨砂背景、亚克力底座、干净桌面建立质感。
避免蓝紫泛光、赛博驾驶舱、密集 HUD、数据雨、假 logo、复杂渐变和过多界面卡片。
```

### 夸张对比冲击风

```text
适合前后对比、工具升级、效果测评、普通结果变高级结果。

必须有清晰左右、前后或上下对比。
左侧可普通、杂乱、暗一点；右侧更高级、更亮、更有秩序。
人物做展示、判断、指向或轻微惊讶。
可以少量使用对勾、叉号、箭头、圈注。
避免过多贴纸、低质网红海报、复杂流程图。
```

### 剧情商业海报风

```text
适合 AI 视频、AI 短剧、剧情演示、电影质感结果、娱乐向内容、复古产品宣传和强叙事封面。

明确前景、中景、背景层次，人物像进入片场、故事世界、复古广告牌或生成结果里。
核心视觉物可用电影帧、片场灯、镜头、场记板、旧屏幕、灯箱、海报、产品盒、生成画面。
标题可以像电影片名、商业海报大字或复古招牌，但必须保持短视频封面可读性。
光线可用主光、轮廓光、片场光、灯箱光、反光和少量雾气。
色彩可用深灰、电影蓝、暖橙、暗红、旧纸白、墨绿、复古蓝。
避免廉价爆炸、魔法阵、脸部过暗、无主题赛博城市、低清脏污、标题不可读。
```

### 高级杂志观点风

```text
适合趋势分析、深度评测、人物观点、商业观察、设计判断。

版心稳定，人物和标题形成封面秩序。
人像强，姿态克制，眼神或手势传达观点。
道具少而准确，例如设备、报告、模型或核心界面。
色彩克制但有焦点。
避免贴纸堆叠、综艺大字、复杂界面、空洞留白。
```

## 4 张竖版封面的差异

首次生成 4 张候选时，用这 4 个方向区分：

1. 前景道具冲击：手机、屏幕、产品、模型或结果物伸向镜头，人物在中下部与道具互动。
2. 结果对比冲击：普通结果和高级结果同时出现，人物做判断、展示或指向。
3. 进入主题世界：人物像进入工具生成的世界、片场、产品空间、工作台或结果场景。
4. 证据特写海报：核心结果、产品界面、模型能力或关键画面占据主体，人物在旁侧做展示或判断；如果无人物，就让核心视觉物成为绝对主角。

如果用户没有上传人脸图，并且 Agent 判断更适合无人物封面，就把以上方向里的「人物动作」改成「核心视觉物动作」：产品伸向镜头、屏幕展示对比、结果画面进入主题世界。

```text
避免：假 logo、水印、二维码、乱码、不可读小字、复杂流程图、长段文字、过多界面卡片、陌生人脸、五官跑偏、元素平均用力、蓝紫科技泛光、赛博霓虹、HUD、数据雨、粒子爆炸、廉价金属字。
```

## 横版延展

当用户选中或确认一张竖版封面后，必须先询问：

「是否需要继续生成横版 `4:3` 和 `16:9`？如果不需要，我就停在竖版；如果想换方向，也可以告诉我新的方向。」

只有用户确认需要横版时，才把已确认的竖版图喂给即梦图片 4.7 模型，生成同系列 `4:3` 和 `16:9`。如果用户不需要，就不要生成横版。如果用户说换一个方向，就根据用户新方向调整后重新生成竖版。

横版不是裁剪，也不是拉伸。必须保持：

- 人物、主角或核心视觉物的设定。
- 主标题和副标题。
- 核心视觉物。
- 一个钩子。
- 已确认的标题字体设计、背景、主角服装和动作、其他元素。
- 风格、色彩、光线、材质和背景世界。
- 如果竖版使用了上传的人脸参考图，横版必须保持同一人物身份和脸部辨识度。
- 如果竖版使用了上传的人脸参考图，横版必须保持人脸原始宽高比例，不能为了适配 `4:3` 或 `16:9` 把脸拉宽、压扁或变形。
- 如果竖版没有人脸参考图，横版只保持普通主角或无人封面的设定，不要突然生成固定真人脸。

横版要重新安排标题、人物和道具，让标题仍然大且可读，人物或核心视觉物还有冲击力。改变比例时优先扩充左右背景、补全场景、移动标题和道具位置、调整排版层级；不要通过拉伸人物或人脸来填满画面。不要裁头、裁标题、换脸、换发型、换色调、换世界观或增加无关对象。

横版提示词模板：

```text
使用即梦图片 4.7 模型，基于已确认的竖版 3:4 封面生成同系列横版 <4:3 或 16:9>。

这不是裁剪、拉伸或换风格，而是同一套封面的横版重构。请保留主标题“<主标题>”、副标题“<副标题>”、标题字体设计“<标题字体设计>”、核心视觉物“<核心视觉物>”、主题钩子“<一个可视化钩子>”、风格“<风格名称>”、背景“<背景世界>”、主角服装和动作“<主角服装和动作>”、其他元素“<其他元素>”。

如果原图使用了上传的人脸参考图，继续保持同一人物身份和脸部辨识度，并保持人脸原始宽高比例和五官比例。禁止为了适配横版把脸横向拉宽、纵向压扁或做任何比例变形。如果原图没有人脸参考图，不要突然生成固定真人脸，保持普通主角或无人封面的设定。

为横版重新安排标题、人物和核心视觉物。标题仍然大且可读，人物或核心视觉物保持冲击力，背景向两侧自然扩展，可以扩充背景、补全场景、调整标题位置和道具排版，但不要拉伸人物身体或人脸。不要裁掉标题、裁掉头部、拉伸变形、换色调、换世界观、增加无关对象、生成假 logo、水印、二维码、乱码和过多小字。
```

## 质量检查

生成后逐张检查：

- 是否严格符合目标比例。
- 标题是否在缩略图大小也能读。
- 是否只有一个明显钩子。
- 如果上传了人脸图，人物是否清楚，参考图身份是否保留。
- 如果上传了人脸图，生成 `3:4`、`4:3` 或 `16:9` 时，人脸宽高比例和五官比例是否自然，没有被拉宽、压扁或变形。
- 如果没上传人脸图，是否没有乱承诺固定真人身份，主角服装和动作是否符合已确认信息。
- 核心视觉物是否能解释主题。
- 背景是否服务主题，而不是抢主体。
- 有没有假 logo、水印、乱码、长段小字、复杂流程图、过多标签。
- 有没有五官跑偏、手指异常、人物遮脸、标题被裁掉。
- 是否落入默认蓝紫科技风、赛博霓虹、金属字和粒子爆炸。

不合格就重生成，不要把失败图当成最终封面。
